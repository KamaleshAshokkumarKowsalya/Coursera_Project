# Module 1 - Project Overview and Requirements Gathering

## Architecture Design

- Frontend: HTML, CSS, JavaScript static portal
- Backend: Java 21, Spring Boot, REST APIs, JWT security
- Database: MySQL for relational data, MongoDB for prescriptions
- Deployment: Docker and GitHub Actions

## User Roles

- Admin: manage doctors, patients, appointments, and reports
- Doctor: view appointments, manage prescriptions, review records
- Patient: book appointments, review prescriptions, manage profile

## User Stories

- As an admin, I want to manage clinic users so that access stays controlled.
- As a doctor, I want to view my appointments so that I can plan visits.
- As a patient, I want to book and update appointments so that I can manage care.
