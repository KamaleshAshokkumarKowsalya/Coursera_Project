# Smart Clinic Management System - User Stories

---

## Admin User Stories

### US-001: Add Doctor
**As an Admin**, I want to add a new doctor so that patients can book appointments.

**Acceptance Criteria**
- Admin can enter doctor's name, specialization, email and phone.
- Doctor is saved successfully.
- Success message is displayed.

**Priority:** High
**Story Points:** 5

---

### US-002: Manage Patients
**As an Admin**, I want to manage patient records so that patient information remains accurate.

**Acceptance Criteria**
- Admin can view all patients.
- Admin can edit patient details.
- Admin can delete a patient.

**Priority:** High
**Story Points:** 5

---

### US-003: Manage Doctors
**As an Admin**, I want to update doctor information so that doctor details remain current.

**Acceptance Criteria**
- Admin can edit doctor details.
- Updated information is saved successfully.

**Priority:** Medium
**Story Points:** 3

---

# Doctor User Stories

### US-004: View Appointments
**As a Doctor**, I want to view my appointment list so that I know which patients I need to see.

**Acceptance Criteria**
- Doctor sees only assigned appointments.
- Appointment date and time are displayed.
- Patient details are visible.

**Priority:** High
**Story Points:** 5

---

### US-005: Create Prescription
**As a Doctor**, I want to create prescriptions for patients so that treatment information is recorded.

**Acceptance Criteria**
- Doctor can enter medicines.
- Doctor can enter dosage.
- Prescription is saved successfully.

**Priority:** High
**Story Points:** 5

---

### US-006: Update Appointment Status
**As a Doctor**, I want to mark appointments as completed or cancelled.

**Acceptance Criteria**
- Doctor can update appointment status.
- Updated status is reflected immediately.

**Priority:** Medium
**Story Points:** 3

---

# Patient User Stories

### US-007: Search Doctor
**As a Patient**, I want to search doctors by specialization or name so that I can book appointments.

**Acceptance Criteria**
- Patient can search by doctor's name.
- Patient can search by specialization.
- Matching doctors are displayed.

**Priority:** High
**Story Points:** 5

---

### US-008: Book Appointment
**As a Patient**, I want to book an appointment with a doctor.

**Acceptance Criteria**
- Patient selects doctor.
- Patient selects appointment date and time.
- Appointment is booked successfully.

**Priority:** High
**Story Points:** 8

---

### US-009: View My Appointments
**As a Patient**, I want to view my appointments so that I can track upcoming visits.

**Acceptance Criteria**
- Patient sees all booked appointments.
- Appointment status is displayed.

**Priority:** Medium
**Story Points:** 3

---

### US-010: Cancel Appointment
**As a Patient**, I want to cancel an appointment if I cannot attend.

**Acceptance Criteria**
- Patient can cancel future appointments.
- Appointment status changes to CANCELLED.

**Priority:** Medium
**Story Points:** 2
